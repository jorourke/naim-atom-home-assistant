# v0.4.0 Architecture Cleanup — Design Spec

## Summary

Refactor the Naim Atom Home Assistant integration to decompose the monolithic `media_player.py` (766 lines) into focused modules, consolidate duplicate WebSocket implementations, fix Issue #5 (state loss after power cut), and fix broken tests. The approach is state-centric: a dedicated state module becomes the single source of truth, the client module owns all device communication, and the entity becomes a thin HA adapter.

## Goals

1. Break `media_player.py` into `state.py`, rewritten `client.py`, and slimmed `media_player.py`
2. Eliminate duplicate WebSocket code (`websocket.py` + `client.py:NaimWebSocketClient` → single implementation in `client.py`)
3. Fix Issue #5: buffer clearing on reconnect + connection-restored state refresh
4. Fix broken/skipped tests that are relevant to the refactored code
5. Update manifest: `iot_class` → `local_push`, `version` → `0.4.0`

## Non-Goals

- New media player features (seek, browse media, play_media)
- Event-driven architecture or event bus
- Full decomposition into many small files
- Changes to `config_flow.py` or `__init__.py`

## File Structure

## Module Design

### state.py — State Management

`NaimPlayerState` is the single source of truth for device state. Both WebSocket messages and HTTP polling write to it through the same `update()` method.

**Key decisions:**

- **Lock protects writes only.** `asyncio.Lock()` on `update()`. Property reads are unlocked — they return simple Python values (atomic in CPython). No need for read-consistency across multiple properties.
- **Debounce is source-aware.** `update()` accepts a `source` parameter (`"user"`, `"websocket"`, `"poll"`). When source is `"user"`, a timestamp is set. When source is `"websocket"` or `"poll"`, fields recently modified by the user (within debounce window) are skipped. This consolidates the scattered timestamp checks currently in the entity.
- **State change callback.** `NaimPlayerState` takes an optional `on_change` callback at init. When `update()` actually changes a value, it fires the callback. The entity passes `self.async_write_ha_state` as this callback, replacing manual `async_write_ha_state()` calls after every mutation.

**Debounce window:** 2.0 seconds for volume and mute (matches current behavior).

**Interface:**

```python
class NaimPlayerState:
    def __init__(self, on_change: Callable[[], None] | None = None,
                 debounce_timeout: float = 2.0):
        ...

    async def update(self, source: str = "poll", **kwargs) -> None:
        """Update state fields. Skips debounced fields when source is not 'user'."""

    # Properties (read-only, no lock):
    # volume, mute, source, state, media_info, duration, position,
    # available, artwork_url, media_title, media_artist, media_album
```

**Also contains:**
- `MediaInfo` dataclass — moved from `media_player.py`, unchanged
- `TransportStates` mapping — moved from `media_player.py`, unchanged

### client.py — Device Communication

Rewritten `NaimClient` consolidates HTTP commands and WebSocket into a single class representing all device communication.

**HTTP side** (carried over from current `NaimApiClient`):
- `get_value()`, `set_value()`, `send_command()`, `select_input()` with existing retry logic and exponential backoff
- New `poll_state()` method — batches the 5 sequential HTTP calls currently scattered in the entity's `async_update()`. Writes results to `NaimPlayerState` with `source="poll"`.

**WebSocket side** (consolidated from two duplicate implementations):
- Single `_socket_listener()` loop with `raw_decode()` incremental JSON parsing
- **Buffer cleared on reconnect** — `self._buffer = ""` at the top of each connection attempt. Fixes Issue #5 where stale JSON fragments from previous connections contaminate new sessions.
- **Connection-restored callback** — when WebSocket reconnects after a disconnect, calls `poll_state()` to do a full state refresh. Ensures accurate state after power cut rather than relying on whatever the first WebSocket message contains.
- **Reconnect jitter** — `sleep(interval + random.uniform(0, interval * 0.3))` to avoid thundering herd with multiple devices.
- **64KB buffer limit** (`MAX_BUFFER_SIZE`) — if exceeded, buffer is cleared and a warning is logged. Prevents unbounded memory growth from corrupted streams.
- WebSocket writes to `NaimPlayerState` with `source="websocket"`.

**HTTP command methods** update state with `source="user"` for optimistic UI updates.

**Interface:**

```python
class NaimClient:
    def __init__(self, hass, host: str, http_port: int, ws_port: int,
                 state: NaimPlayerState):
        ...

    # HTTP commands
    async def set_volume(self, volume: int) -> None
    async def set_mute(self, mute: bool) -> None
    async def set_power(self, on: bool) -> None
    async def send_playback_command(self, cmd: str) -> None
    async def select_input(self, input_id: str) -> None
    async def poll_state(self) -> None

    # WebSocket lifecycle
    async def start_websocket(self) -> None
    async def stop_websocket(self) -> None
```

### media_player.py — HA Entity Adapter

The slimmed entity class contains only HA integration glue:

- **Init:** Creates `NaimPlayerState` (with `async_write_ha_state` as callback) and `NaimClient`. Starts WebSocket via `asyncio.create_task()`.
- **Properties:** Delegate to `self._state` — volume_level, is_volume_muted, source, state, media_title, media_artist, media_album_name, media_image_url, media_duration, media_position.
- **Commands:** Delegate to `self._client` — async_volume_up, async_volume_down, async_set_volume_level, async_mute_volume, async_media_play, async_media_pause, async_media_next_track, async_media_previous_track, async_turn_on, async_turn_off, async_select_source.
- **`async_update()`:** Single call to `self._client.poll_state()`.
- **Lifecycle:** `async_will_remove_from_hass()` stops WebSocket.

No debounce logic, no HTTP calls, no JSON parsing, no buffer management.

## Issue #5 Fix: State Loss After Power Cut

**Problem:** When the Naim device loses power, the WebSocket disconnects. On reconnect, stale data in the buffer can produce corrupted JSON, and the first WebSocket message may not contain full state.

**Fix (two parts):**
1. **Buffer clearing:** `self._buffer = ""` at the start of each reconnection attempt in `_socket_listener()`. Prevents stale fragments from contaminating the new session.
2. **Connection-restored refresh:** After WebSocket reconnects, `_socket_listener()` calls `poll_state()` to fetch full device state via HTTP. This ensures HA has accurate state immediately, not just whatever the first WebSocket push contains.
